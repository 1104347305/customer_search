### 2. ES索引配置更新

**文件**: `scripts/init_elasticsearch.py`

**新增配置**:
```python
"filter": {
    "pinyin_filter": {
        "type": "pinyin",
        "keep_first_letter": True,           # 保留首字母 (z, s)
        "keep_separate_first_letter": False, # 不分离首字母
        "keep_full_pinyin": True,            # 保留完整拼音 (zhang, san)
        "keep_original": True,               # 保留原始中文 (张, 三)
        "limit_first_letter_length": 16,     # 首字母长度限制
        "lowercase": True,                   # 转小写
        "remove_duplicated_term": True       # 去重
    }
}
```

**分词效果**:
- 输入: "张三"
- 输出tokens: ["zhang", "san", "zhangsan", "张", "三", "z", "s"]

### 3. 查询构建器优化

**文件**: `app/services/query_builder.py` (第238-254行)

**修改前**:
```python
# 姓名：中文模糊匹配（由于拼音插件未安装，暂时只用中文匹配）
if field == "name":
    return {
        "bool": {
            "should": [
                {"term": {"name.keyword": {"value": value, "boost": 100}}},
                {"match": {"name": {"query": value, "fuzziness": "AUTO", "boost": 50}}},
                {"prefix": {"name": {"value": value, "boost": 20}}},
                {"wildcard": {"name": {"value": f"*{value}*", "boost": 1}}}
            ],
            "minimum_should_match": 1
        }
    }
```

**修改后**:
```python
# 姓名：中文+拼音模糊匹配
if field == "name":
    return {
        "bool": {
            "should": [
                {"term": {"name.keyword": {"value": value, "boost": 100}}},
                {"match": {"name": {"query": value, "fuzziness": "AUTO", "boost": 50}}},
                {"match": {"name.pinyin": {"query": value, "fuzziness": "AUTO", "boost": 50}}},  # 新增
                {"prefix": {"name": {"value": value, "boost": 20}}},
                {"wildcard": {"name": {"value": f"*{value}*", "boost": 1}}}
            ],
            "minimum_should_match": 1
        }
    }
```

**关键变化**:
- 新增 `name.pinyin` 字段的模糊匹配
- 使用 `fuzziness=AUTO` 支持拼音相似度计算
- boost权重设为50，与中文匹配权重相同

## 技术细节

### 拼音相似度算法

使用Elasticsearch的Levenshtein距离（编辑距离）计算拼音相似度：

```
"shan" vs "san"   : 编辑距离=1 (删除h) → 高分
"shan" vs "shuai" : 编辑距离=3 (替换+插入) → 中分
"shan" vs "peng"  : 编辑距离=4 (完全不同) → 低分
```

### Fuzziness=AUTO规则

| 字符串长度 | 允许编辑距离 |
|-----------|-------------|
| 0-2 | 0 (必须精确匹配) |
| 3-5 | 1 (允许1个字符差异) |
| >5  | 2 (允许2个字符差异) |

**示例**:
- "san" (3字符) → 允许编辑距离=1 → 可匹配"shan"
- "zhangsan" (8字符) → 允许编辑距离=2 → 可匹配"zhansan", "zhangsan"

## 已知限制

1. **多音字处理**: 拼音插件会生成所有可能的读音，可能导致误匹配
   - 示例: "重庆"可能匹配"崇庆"

2. **拼音首字母**: 首字母搜索可能返回大量结果
   - 示例: "zs"会匹配所有姓张、赵、周、郑等的客户

3. **性能开销**: 拼音分析增加约20%的索引大小和导入时间
