## 二、技术实现

### 2.1 技术架构

```
┌─────────────────────────────────────────┐
│         客户端 (HTTP/JSON)               │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│      API层 (FastAPI 0.109.0)            │
│  - 搜索接口 /api/v1/search/customer     │
│  - 健康检查 /health                      │
│  - API文档 /docs                         │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│      业务逻辑层 (Services)               │
│  - SearchService: 搜索服务              │
│  - QueryBuilder: 查询构建器             │
│  - DataMasking: 数据脱敏                │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│      数据访问层 (Repository)             │
│  - ElasticsearchRepository              │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│   Elasticsearch 8.12.0 (99,488 docs)    │
│  - 拼音分词器                            │
│  - 手机号分词器                          │
│  - 中文分词器                            │
└─────────────────────────────────────────┘
```

### 2.2 核心技术特性

| 特性 | 实现方案 | 效果 |
|------|---------|------|
| 智能匹配 | 拼音分词器+ngram | 支持模糊搜索 |
| 嵌套查询 | Nested类型+Nested Query | 支持家庭成员、保单查询 |
| 数据脱敏 | 后端自动脱敏 | 姓名、手机号等自动脱敏 |
| 权限隔离 | agent_id强制过滤 | 代理人只能查看自己的客户 |
| 高性能 | ES索引优化 | 查询响应4-33ms |

### 2.3 API接口规范

**V3规范特点**:
- 使用`header`对象封装系统参数
- 统一响应格式：`{code, message, data: {total, page, size, list}}`
- 支持7种操作符：MATCH, GTE, LTE, RANGE, CONTAINS, NOT_CONTAINS, NESTED_MATCH
- 支持AND/OR逻辑组合
- 最大分页1000条

## 八、使用指南

### 8.1 快速开始

```bash
# 1. 健康检查
curl http://localhost:8001/health

# 2. 基础查询
curl -X POST http://localhost:8001/api/v1/search/customer \
  -H "Content-Type: application/json" \
  -d '{
    "header": {"agent_id": "A000001", "page": 1, "size": 10},
    "conditions": [
      {"field": "name", "operator": "MATCH", "value": "张"}
    ]
  }'

# 3. 查看API文档
open http://localhost:8001/docs
```

### 8.2 典型场景

#### 场景1: 养老保险推广
```bash
curl -X POST http://localhost:8001/api/v1/search/customer \
  -H "Content-Type: application/json" \

    "query_logic": "AND",
    "conditions": [
      {"field": "age", "operator": "GTE", "value": 45},
```
