## 详细修复内容

### 修复1: RANGE操作符格式兼容性 ✅

**问题**: 只支持数组格式 `[min, max]`，不支持字典格式 `{"min": x, "max": y}`

**影响测试用例**:
- P0-1.5: 查找35岁有小朋友还没配置重疾险的客户
- P1-3.1: 查找二十多岁刚结婚没买保险的客户
- P1-5.2: 姓氏+年龄+地址组合搜索（年龄部分）

**修复代码** (`app/services/query_builder.py` 第94-111行):
```python
elif operator == OperatorEnum.RANGE:
    # 支持数组格式: [min, max]
    if isinstance(value, (list, tuple)) and len(value) == 2:
        return {"range": {field: {"gte": value[0], "lte": value[1]}}}

    # 支持字典格式: {"min": x, "max": y}
    elif isinstance(value, dict) and ('min' in value or 'max' in value):
        range_query = {}
        if 'min' in value:
            range_query['gte'] = value['min']
        if 'max' in value:
            range_query['lte'] = value['max']
        return {"range": {field: range_query}}

    else:
        raise InvalidQueryError(
            f"RANGE operator requires [min, max] or {{'min': x, 'max': y}} format, got: {value}"
        )
```

**验证结果**:
```bash
# 数组格式
{"field": "age", "operator": "RANGE", "value": [30, 40]}  # ✅ 工作

# 字典格式
{"field": "age", "operator": "RANGE", "value": {"min": 30, "max": 40}}  # ✅ 工作
```

**业务价值**:
- 提升API容错性
- 支持更灵活的查询格式
- 3个测试用例从失败变为通过

### 修复2: NESTED_MATCH支持range查询 ✅

**问题**: 只支持term查询（精确匹配），不支持range查询（范围匹配）

**错误信息**: `[term] query does not support [min]`

**影响场景**:
- 家庭成员年龄范围查询
- 保单金额范围查询
- 所有嵌套对象的范围查询

**修复代码** (`app/services/query_builder.py` 第123-160行):
```python
elif operator == OperatorEnum.NESTED_MATCH:
    if not condition.nested_path:
        raise InvalidQueryError("NESTED_MATCH requires nested_path parameter")

    # 构建嵌套查询的内部查询
    nested_query = None

    # 如果value是字典且包含min/max，使用range查询
    if isinstance(value, dict) and ('min' in value or 'max' in value):
        range_query = {}
        if 'min' in value:
            range_query['gte'] = value['min']
        if 'max' in value:
            range_query['lte'] = value['max']
        nested_query = {"range": {f"{condition.nested_path}.{field}": range_query}}

    # 如果value是None，查询字段不存在或为空
    elif value is None:
        nested_query = {
            "bool": {
                "should": [
                    {"bool": {"must_not": [{"exists": {"field": f"{condition.nested_path}.{field}"}}]}},
                    {"term": {f"{condition.nested_path}.{field}": ""}}
                ],
                "minimum_should_match": 1
            }
        }

    # 否则根据字段类型选择查询方式
    else:
        # 对于id_number等文本字段，使用match查询支持部分匹配
        if field == "id_number":
            nested_query = {"match": {f"{condition.nested_path}.{field}": value}}
        # 其他字段使用term查询（精确匹配）
        else:
            nested_query = {"term": {f"{condition.nested_path}.{field}": value}}

    return {
        "nested": {
            "path": condition.nested_path,
            "query": nested_query
        }
    }
```

**支持的查询类型**:

1. **Range查询** - 范围匹配
```json
{
  "field": "age",
  "operator": "NESTED_MATCH",
  "value": {"min": 6, "max": 12},
  "nested_path": "family_members"
}
```

2. **Null查询** - 字段不存在或为空
```json
{
  "field": "has_insurance",
  "operator": "NESTED_MATCH",
  "value": null,
  "nested_path": "family_members"
}
```

3. **Match查询** - 部分匹配（id_number字段）
```json
{
  "field": "id_number",
  "operator": "NESTED_MATCH",
  "value": "19980324",
  "nested_path": "certificates"
}
```

4. **Term查询** - 精确匹配（其他字段）
```json
{
  "field": "relationship",
  "operator": "NESTED_MATCH",
  "value": "子女",
  "nested_path": "family_members"
}
```

**业务价值**:
- 支持家庭成员年龄范围查询（如6-12岁小学生）
- 支持保单金额范围查询
- 为未来数据扩展做好准备

### 修复3: NESTED_MATCH身份证号搜索 ✅

**问题**: 使用NESTED_MATCH查询身份证号返回空结果

**用户查询**:
```json
{
  "field": "id_number",
  "operator": "NESTED_MATCH",
  "value": "19980324",
  "nested_path": "certificates"
}
```

**根本原因**:
- `certificates.id_number` 是text类型，配置了phone_analyzer（ngram分词）
- 原实现使用term查询，不会触发分词器
- term查询无法实现部分匹配

**修复方案**:
在NESTED_MATCH操作符中，对id_number字段使用match查询而非term查询

**验证结果**:
```bash
# 按出生日期搜索
{"value": "19980324"}  # ✅ 返回29条记录

# 按地区代码搜索
{"value": "110101"}  # ✅ 返回91条记录（北京东城区）

# 按完整身份证号搜索
{"value": "110101198711052832"}  # ✅ 返回1条记录
```

**支持的业务场景**:
1. **按出生日期查询** - 生日营销
2. **按地区代码查询** - 区域营销
3. **按身份证号片段查询** - 模糊查询
4. **按完整身份证号查询** - 精确查找

**业务价值**:
- 支持生日营销（查找特定日期出生的客户）
- 支持区域营销（查找特定地区的客户）
- 提升用户体验（支持模糊查询）
