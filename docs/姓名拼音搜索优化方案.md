# 姓名拼音搜索优化方案

**问题**: 搜索"张山"时，期望"张三"排在"张鹏"前面（因为拼音更相近：shan vs san）

**当前状态**: "张三"和"张鹏"评分相同，无法区分拼音相似度

---

## 根本原因

ES索引中的 `name.pinyin` 字段使用的 `pinyin_analyzer` 配置不正确：

```json
{
  "pinyin_analyzer": {
    "type": "custom",
    "tokenizer": "standard",  // ❌ 只是标准分词，没有拼音转换
    "filter": ["lowercase"]
  }
}
```

**实际效果**：
- 输入："张三" → 分词结果：["张", "三"]
- 应该是："张三" → 分词结果：["zhang", "san", "zhangsan"]

---

## 解决方案：安装拼音插件

### 步骤1: 安装elasticsearch-analysis-pinyin插件

```bash
# 1. 查看ES版本
curl http://localhost:9200

# 2. 进入ES容器（如果使用Docker）
docker exec -it <es_container_id> bash

# 3. 安装拼音插件（根据ES版本选择）
# ES 8.12.0
./bin/elasticsearch-plugin install https://github.com/medcl/elasticsearch-analysis-pinyin/releases/download/v8.12.0/elasticsearch-analysis-pinyin-8.12.0.zip

# 4. 重启ES
docker restart <es_container_id>
```

### 步骤2: 重新配置索引

**新的拼音分析器配置**：

```json
{
  "settings": {
    "analysis": {
      "analyzer": {
        "pinyin_analyzer": {
          "type": "custom",
          "tokenizer": "standard",
          "filter": ["pinyin_filter", "lowercase"]
        }
      },
      "filter": {
        "pinyin_filter": {
          "type": "pinyin",
          "keep_first_letter": true,
          "keep_separate_first_letter": false,
          "keep_full_pinyin": true,
          "keep_original": true,
          "limit_first_letter_length": 16,
          "lowercase": true,
          "remove_duplicated_term": true
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "name": {
        "type": "text",
        "analyzer": "standard",
        "fields": {
          "keyword": {
            "type": "keyword"
          },
          "pinyin": {
            "type": "text",
            "analyzer": "pinyin_analyzer"
          }
        }
      }
    }
  }
}
```

### 步骤3: 重建索引

```bash
# 1. 删除旧索引
curl -X DELETE "http://localhost:9200/customers"

# 2. 创建新索引（使用新配置）
python scripts/init_elasticsearch.py

# 3. 重新导入数据
python scripts/import_data.py
```

### 步骤4: 验证效果

```bash
# 测试拼音分析器
curl -X POST "http://localhost:9200/customers/_analyze" -H "Content-Type: application/json" -d '{
  "field": "name.pinyin",
  "text": "张三"
}'

# 期望结果：["zhang", "san", "zhangsan", "张", "三"]
```

## 临时方案（不推荐）

如果暂时无法安装拼音插件，可以：

1. **接受当前排序** - "张三"和"张鹏"评分相同是正常的
2. **前端排序** - 在前端使用JavaScript拼音库进行二次排序
3. **应用层排序** - 在Python中使用pypinyin库计算相似度并重新排序
