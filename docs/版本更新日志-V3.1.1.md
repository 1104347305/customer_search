## 🐛 问题修复

### 问题描述

用户查询返回空结果：
```json
{
  "header": {"agent_id": "A000001", "page": 1, "size": 10},
  "query_logic": "AND",
  "conditions": [
    {"field": "age", "operator": "RANGE", "value": {"min": 35, "max": 35}},
    {"field": "family_members.relationship", "operator": "CONTAINS", "value": "子女"},
    {"field": "life_liability_type", "operator": "NOT_CONTAINS", "value": "重疾"}
  ]
}
```

**根本原因**：
- `CONTAINS` 和 `NOT_CONTAINS` 操作符不支持嵌套字段的自动检测
- 只有 `MATCH` 操作符具备嵌套字段自动检测功能
- 用户必须使用 `NESTED_MATCH` 操作符并手动指定 `nested_path`，使用门槛高

### 修复方案

在 `app/services/query_builder.py` 中为 `CONTAINS` 和 `NOT_CONTAINS` 操作符添加嵌套字段自动检测逻辑：

**修改前**（第159-167行）：
```python
elif operator == OperatorEnum.CONTAINS:
    # Only add .keyword suffix for text fields that have it
    term_field = f"{field}.keyword" if field in QueryBuilder.TEXT_FIELDS_WITH_KEYWORD else field
    return {"term": {term_field: value}}

elif operator == OperatorEnum.NOT_CONTAINS:
    # Only add .keyword suffix for text fields that have it
    term_field = f"{field}.keyword" if field in QueryBuilder.TEXT_FIELDS_WITH_KEYWORD else field
    return {"bool": {"must_not": [{"term": {term_field: value}}]}}
```

**修改后**：
```python
elif operator == OperatorEnum.CONTAINS:
    # 自动检测是否为嵌套字段
    if '.' in field:
        parts = field.split('.', 1)
        potential_path = parts[0]

        # 如果是嵌套字段，使用nested查询
        if potential_path in QueryBuilder.NESTED_PATHS:
            return QueryBuilder._build_nested_query(field, value, None)

    # 普通字段：Only add .keyword suffix for text fields that have it
    term_field = f"{field}.keyword" if field in QueryBuilder.TEXT_FIELDS_WITH_KEYWORD else field
    return {"term": {term_field: value}}

elif operator == OperatorEnum.NOT_CONTAINS:
    # 自动检测是否为嵌套字段
    if '.' in field:
        parts = field.split('.', 1)
        potential_path = parts[0]

        # 如果是嵌套字段，使用nested查询 + must_not
        if potential_path in QueryBuilder.NESTED_PATHS:
            nested_query = QueryBuilder._build_nested_query(field, value, None)
            return {"bool": {"must_not": [nested_query]}}

    # 普通字段：Only add .keyword suffix for text fields that have it
    term_field = f"{field}.keyword" if field in QueryBuilder.TEXT_FIELDS_WITH_KEYWORD else field
    return {"bool": {"must_not": [{"term": {term_field: value}}]}}
```

## ✨ 功能增强

### 1. 嵌套字段自动检测

**支持的操作符**：
- ✅ `MATCH` - 已支持（V3.0）
- ✅ `CONTAINS` - 新增支持（V3.1.1）
- ✅ `NOT_CONTAINS` - 新增支持（V3.1.1）

**支持的嵌套路径**：
- `family_members.*` - 家庭成员
- `policies.*` - 保单信息
- `certificates.*` - 证件信息
- `benefits.member_info.*` - 会员权益
- `benefits.pingan_info.*` - 平安权益

### 2. 简化查询语法

**修复前**（必须使用 NESTED_MATCH）：
```json
{
  "field": "relationship",
  "operator": "NESTED_MATCH",
  "value": "子女",
  "nested_path": "family_members"
}
```

**修复后**（可以直接使用 CONTAINS）：
```json
{
  "field": "family_members.relationship",
  "operator": "CONTAINS",
  "value": "子女"
}
```

## ⚠️ 注意事项

1. **自动检测仅适用于已定义的嵌套路径**
   - 系统会检查字段名的第一部分是否在 `NESTED_PATHS` 中
   - 未定义的路径不会触发自动检测

2. **NESTED_MATCH 仍然推荐用于复杂查询**
   - 如需使用 range 查询嵌套字段的数值
   - 如需更精细的控制查询行为

3. **点号表示法的限制**
   - 字段名本身不能包含点号
   - 仅支持一级嵌套路径（如 `family_members.relationship`）
