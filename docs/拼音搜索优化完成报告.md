## 问题回顾

**用户需求**: 搜索"张山"时，期望"张三"排在"张鹏"前面，因为拼音更相近（shan vs san）

**原问题**: "张三"和"张鹏"评分相同，无法区分拼音相似度

    }

## 技术原理

### 拼音相似度计算

Elasticsearch的拼音插件会将中文转换为拼音tokens：

- "张三" → ["zhang", "san", "zhangsan", "张", "三", "z", "s"]
- "张山" → ["zhang", "shan", "zhangshan", "张", "山", "z", "s"]
- "张鹏" → ["zhang", "peng", "zhangpeng", "张", "鹏", "z", "p"]

当搜索"张山"时，使用`fuzziness=AUTO`的match查询：
- "san" vs "shan": 编辑距离=1（非常相似）→ 高分
- "peng" vs "shan": 编辑距离=4（不相似）→ 低分

### 评分权重

```
精确匹配 (name.keyword)     : boost=100
中文模糊匹配 (name)          : boost=50
拼音模糊匹配 (name.pinyin)   : boost=50
前缀匹配 (prefix)            : boost=20
包含匹配 (wildcard)          : boost=1
```
