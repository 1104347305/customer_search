# 客户搜索系统 - 项目说明书

## 📋 项目概述

### 项目名称
保险代理人客户搜索系统 (Customer Search System)

### 项目简介
基于 Elasticsearch 的高性能客户搜索系统，专为保险代理人设计，支持多维度组合查询、智能匹配、嵌套对象搜索等复杂业务场景。系统采用 FastAPI 框架构建 RESTful API，提供生产级的数据检索服务。

### 项目背景
- **业务需求**: 支持 1000 个代理人管理 10 万客户数据的高性能检索
- **核心痛点**: 传统数据库难以支持复杂的多维度组合查询和模糊匹配
- **解决方案**: 采用 Elasticsearch 搜索引擎，实现毫秒级响应的智能搜索

### 项目目标
- ✅ 查询响应时间 < 50ms
- ✅ 支持 10 万级数据量
- ✅ 支持复杂组合查询
- ✅ 数据安全与权限隔离
- ✅ 生产级代码质量

---

## 🎯 核心功能

### 1. 智能搜索
- **姓名搜索**: 支持拼音模糊匹配（如"zhang"可匹配"张三"）
- **手机号搜索**: 支持片段匹配（如"138"可匹配所有138开头的号码）
- **地址搜索**: 支持中文分词匹配
- **枚举值搜索**: 精确匹配客户分类、温度等枚举字段

### 2. 多维度组合查询
- **7种操作符**: MATCH、GTE、LTE、RANGE、CONTAINS、NOT_CONTAINS、NESTED_MATCH
- **逻辑组合**: 支持 AND/OR 逻辑
- **嵌套查询**: 支持保单、家庭成员等嵌套对象查询
- **排序功能**: 支持多字段排序

### 3. 数据安全
- **自动脱敏**: 姓名、手机号、身份证、保单号等敏感信息自动脱敏
- **权限隔离**: 基于 agent_id 的数据权限控制，代理人只能查看自己的客户
- **数据完整性**: 返回完整的客户信息（100+ 字段）

### 4. 高性能
- **查询响应**: 4-33ms 平均响应时间
- **大分页支持**: 单次最多返回 1000 条记录
- **批量索引**: 高效的数据导入机制

## 📡 API 接口规范 (V3)

### 接口信息

- **API 地址**: http://localhost:8001
- **接口路径**: /api/v1/search/customer
- **请求方法**: POST
- **数据格式**: application/json
- **API 文档**: http://localhost:8001/docs

### 请求格式

```json
{
  "header": {
    "agent_id": "A000001",        // 代理人ID（必填）
    "page": 1,                    // 页码（默认1）
    "size": 10                    // 每页数量（默认10，最大1000）
  },
  "query_logic": "AND",           // 条件逻辑：AND/OR（默认AND）
  "conditions": [                 // 查询条件列表（必填）
    {
      "field": "name",            // 字段名
      "operator": "MATCH",        // 操作符
      "value": "张"               // 查询值
    }
  ],
  "sort": [                       // 排序规则（可选）
    {
      "field": "age",             // 排序字段
      "order": "desc"             // 排序方向：asc/desc
    }
  ]
}
```

### 响应格式

```json
{
  "code": 200,                    // 响应码（200=成功）
  "message": "success",           // 响应消息
  "data": {
    "total": 100,                 // 总记录数
    "page": 1,                    // 当前页码
    "size": 10,                   // 每页数量
    "list": [                     // 数据列表
      {
        "name": "张*",            // 脱敏后的姓名
        "mobile_phone": "138****5678",
        "age": 35,
        "customer_value": "A1类客户",
        // ... 其他字段
      }
    ]
  }
}
```

### 支持的操作符

| 操作符 | 说明 | 适用字段 | 示例 |
|--------|------|----------|------|
| MATCH | 智能匹配 | 姓名、手机号、地址、枚举值 | `{"field": "name", "operator": "MATCH", "value": "张"}` |
| GTE | 大于等于 | 数值字段（年龄、收入、保费） | `{"field": "age", "operator": "GTE", "value": 30}` |
| LTE | 小于等于 | 数值字段 | `{"field": "age", "operator": "LTE", "value": 60}` |
| RANGE | 区间查询 | 数值字段 | `{"field": "age", "operator": "RANGE", "value": {"min": 30, "max": 60}}` |
| CONTAINS | 包含 | 数组字段、文本字段 | `{"field": "held_product_category", "operator": "CONTAINS", "value": "重疾险"}` |
| NOT_CONTAINS | 不包含 | 数组字段 | `{"field": "held_product_category", "operator": "NOT_CONTAINS", "value": "养老保险"}` |
| NESTED_MATCH | 嵌套查询 | 嵌套对象（保单、家庭成员） | `{"field": "status", "operator": "NESTED_MATCH", "value": "缴费有效", "nested_path": "policies"}` |

## 🔐 数据安全与权限

### 数据脱敏规则

系统自动对敏感信息进行脱敏处理，确保数据安全：

| 字段类型 | 原始值 | 脱敏后 | 脱敏规则 |
|---------|--------|--------|----------|
| 姓名 | 张三 | 张* | 保留姓氏，名字用*替代 |
| 手机号 | 13812345678 | 138****5678 | 保留前3位和后4位 |
| 身份证号 | 110101199001011234 | 110101********1234 | 保留前6位和后4位 |
| 保单号 | 9200111115555555 | 920011****5555 | 保留前6位和后4位 |
| 邮箱 | zhangsan@example.com | z***@example.com | 保留首字母和域名 |

### 权限控制

- **代理人隔离**: 所有查询自动添加 `agent_id` 过滤条件
- **数据边界**: 代理人只能查看自己的客户数据
- **无跨域访问**: 无法通过任何方式查询其他代理人的客户

### 安全特性

- ✅ 请求参数验证（Pydantic）
- ✅ SQL注入防护（使用ES DSL）
- ✅ 敏感信息自动脱敏
- ✅ 权限强制隔离
- ✅ 错误信息不泄露敏感数据

### Elasticsearch 索引配置

#### 分词器配置

1. **拼音分词器 (pinyin_analyzer)**
   - 用途：姓名拼音搜索
   - 示例：输入"zhang"可匹配"张三"

2. **手机号分词器 (phone_analyzer)**
   - 用途：手机号片段匹配
   - 配置：ngram (min=3, max=11)
   - 示例：输入"138"可匹配所有138开头的号码

3. **中文分词器 (ik_max_word)**
   - 用途：地址、备注等中文文本分词
   - 示例：输入"北京朝阳"可匹配"北京市朝阳区"

## 📈 性能优化

### 已实现的优化

1. **索引优化**
   - 3个主分片，1个副本
   - 合理的字段映射类型
   - 多字段索引（text + keyword）

2. **查询优化**
   - 智能选择分词器
   - 避免全表扫描
   - 使用 filter context 提升性能

3. **批量操作**
   - 数据导入使用 bulk API
   - 批次大小：500条/批

4. **连接池管理**
   - Elasticsearch 客户端连接池
   - 连接复用

5. **数据脱敏**
   - 仅在返回时脱敏
   - 不影响查询性能

## 🐛 故障排查

### 常见问题

#### 1. Elasticsearch 连接失败

**症状**: API启动时报错 "Connection refused"

**解决方案**:
```bash
# 检查 ES 是否启动
curl http://localhost:9200

# 如未启动，启动 ES
cd docker
docker-compose up -d elasticsearch

# 查看 ES 日志
docker logs customer_search_es
```

#### 2. 数据导入失败

**症状**: import_data.py 报错

**解决方案**:
```bash
# 检查索引是否存在
curl http://localhost:9200/customers

# 重新初始化索引
python scripts/init_elasticsearch.py

# 重新导入数据
python scripts/import_data.py
```

#### 3. API 查询无结果

**症状**: 查询返回 total=0

**可能原因**:
- agent_id 不存在
- 查询条件过于严格
- 数据未正确导入

**解决方案**:
```bash
# 检查数据总量
curl http://localhost:9200/customers/_count

# 检查特定代理人的数据
curl -X POST http://localhost:9200/customers/_search \
  -H "Content-Type: application/json" \
  -d '{"query": {"term": {"agent_id": "A000001"}}}'
```

#### 4. 响应时间过长

**症状**: 查询响应时间 > 100ms

**可能原因**:
- 查询条件复杂
- 数据量过大
- ES 资源不足

**解决方案**:
```bash
# 检查 ES 集群状态
curl http://localhost:9200/_cluster/health

# 优化查询条件
# 减少返回字段
# 使用更精确的查询条件
```

#### 5. V3 格式错误

**症状**: "Field required" 错误

**原因**: 使用了旧版 API 格式

**解决方案**:
```json
// ❌ 旧格式（不支持）
{
  "agent_id": "A000001",
  "conditions": [...]
}

// ✅ 新格式（V3）
{
  "header": {
    "agent_id": "A000001",
    "page": 1,
    "size": 10
  },
  "conditions": [...]
}
```
